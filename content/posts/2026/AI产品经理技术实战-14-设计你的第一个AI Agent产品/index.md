## 一、选题：找到一个值得做的AI产品机会

### 1.1 选题的三个标准

不是所有想法都值得做。一个好的AI产品想法需要满足三个标准：

**标准一：真实需求**

用户真的需要这个功能吗？还是你"觉得"他们需要？

验证方法：
- 和用户聊10个以上
- 看现有解决方案的痛点
- 分析用户行为数据

**标准二：AI能解决**

这个问题适合用AI解决吗？还是传统方案就够了？

用第二章的决策树判断。

**标准三：有数据支撑**

你有足够的数据训练或检索吗？

### 1.2 选题案例：AI会议纪要助手

**需求发现：**

我和10个职场人士聊天，发现他们共同的痛点是：
- 开会多，记录忙不过来
- 会后忘记会议内容和行动项
- 会议记录整理花30分钟以上

**AI适用性判断：**

- 是信息处理问题吗？是（理解语音→生成纪要）
- 规则能穷举吗？不能（会议内容千变万化）
- 有足够数据吗？有（会议录音+文字转录）
- 容错率足够吗？是（纪要写错了可以手动改）

→ 适合用AI

**数据可行性：**

- 有会议录音数据
- 有文字转录API可用
- 有历史会议纪要作为训练数据

→ 数据可行

**结论：✅ 值得做**

## 二、需求分析

### 2.1 用户画像

```
用户A：项目经理，30岁
- 每天开3-5个会
- 需要整理会议纪要并发给团队
- 痛点：整理纪要花30分钟，经常漏掉行动项

用户B：产品经理，28岁
- 每周参加2-3个需求评审会
- 需要记录决策和待办事项
- 痛点：会上没时间记，会后回忆不起来

用户C：实习生，23岁
- 第一次参加正式会议
- 听不懂专业术语
- 痛点：跟不上会议节奏，会后看不懂纪要
```

### 2.2 核心需求

```
P0（必须有）：
- 实时语音转文字
- 自动生成会议纪要
- 提取行动项（谁在什么时候做什么）

P1（应该有）：
- 会议摘要（300字以内）
- 关键词提取
- 多语言支持

P2（可以有）：
- 会议问答（会后问AI"会上讨论了XX吗"）
- 智能提醒（行动项到期前提醒相关人）
```

### 2.3 效果指标

```
- 语音转文字准确率：≥95%
- 纪要生成准确率：≥90%
- 行动项提取准确率：≥85%
- 用户满意度：≥80%
- 单次会议处理时间：<5分钟
```

### 3.2 核心流程

```
用户点击"开始录音"
       │
       ▼
语音实时转文字（Whisper模型）
       │
       ▼
文字流式输入到纪要Agent
       │
       ▼
纪要Agent处理：
  1. 识别会议主题
  2. 提取关键讨论点
  3. 识别行动项（谁+做什么+什么时候）
  4. 生成300字摘要
       │
       ▼
输出会议纪要：
  - 会议主题
  - 关键讨论点
  - 行动项列表
  - 300字摘要
       │
       ▼
用户编辑确认
       │
       ▼
发送纪要给参会人
```

### 3.3 Prompt设计

**纪要Agent的Prompt：**

```
你是一个专业的会议纪要助手。

请阅读以下会议录音转写的文字，完成以下任务：

1. 提取会议主题（一句话）
2. 提取关键讨论点（3-5个，每个不超过50字）
3. 提取行动项（谁在什么时候做什么）
4. 生成300字以内的会议摘要

输出格式（JSON）：
{
  "topic": "会议主题",
  "key_points": ["要点1", "要点2", ...],
  "action_items": [
    {"who": "人名", "what": "做什么", "deadline": "时间"}
  ],
  "summary": "300字以内的摘要"
}

约束条件：
- 如果某人没有明确的时间承诺，deadline填"待定"
- 如果无法识别行动项，action_items为空数组
- 只基于会议内容生成，不要添加会议中没有的信息
```

## 四、技术方案

### 4.1 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 语音转文字 | Whisper（开源） | 免费、准确率高、支持多语言 |
| 纪要生成 | GPT-4o | 理解能力强、JSON输出稳定 |
| 向量数据库 | Pinecone | 托管服务、开箱即用 |
| 部署 | 云端 | 初期成本低、弹性伸缩 |

### 4.2 成本估算

```
假设：每天1000场会议，每场平均30分钟

语音转写：
- Whisper开源免费
- 服务器成本：约¥500/月

纪要生成：
- 每场会议转写文字约5000字 ≈ 7000 Token
- GPT-4o输入：¥0.03/千Token → 7000 Token ≈ ¥0.21
- GPT-4o输出：约500字 ≈ 700 Token → ¥0.04
- 每场会议模型成本：约¥0.25
- 每天1000场：¥250/天 ≈ ¥7500/月

总成本：约¥8000/月

定价策略：
- 个人版：¥29/月（10场/月）
- 团队版：¥199/月（不限场次）
- 企业版：¥999/月（定制+私有化）

盈亏平衡点：约400个团队版用户
```

## 五、上线计划

### 5.1 灰度策略

```
Phase 1（第1周）：内部测试
- 团队内部使用
- 收集Bug和Bad Case
- 优化Prompt

Phase 2（第2-3周）：Beta用户
- 邀请100个Beta用户
- 免费使用，换取反馈
- 收集使用数据

Phase 3（第4周）：公开上线
- 全量开放
- 监控效果指标
- 持续迭代
```

### 5.2 监控指标

```
每日监控：
- 会议处理量
- 语音转写准确率
- 纪要生成成功率
- 用户满意度

每周分析：
- Bad Case分类统计
- 用户反馈关键词趋势
- 成本变化趋势

每月评估：
- DAU/MAU
- 留存率
- 付费转化率
```
